# BackDesk 更新说明 (Release Notes)

## 🏷️ 版本号: v0.2.9

BackDesk `v0.2.9` 是一次聚焦 Intel 机型和 Dock 边缘区域体验的稳定性更新。

### 点击触发稳定性
*   修复 Dock 两侧预留空白区域在部分坐标下被 AX 误识别为 `com.apple.dock` 后提前拦截的问题；现在会继续执行窗口遮挡检测，确保空白处可以单击展示桌面。
*   保留真实 Dock 图标、Dock 窗口和被应用窗口占用区域的拦截逻辑，避免点击 Dock 或窗口 UI 时误触发。
*   恢复 macOS 13 及以下系统的稳定展示桌面路径：单击壁纸时直接调用 Mission Control 并传入 Show Desktop 参数，避免仅发送系统快捷键在部分 Intel 机器上无效。
*   补齐 x86_64 调试包与发布包构建验证，方便在 Intel 机器上排查和分发。

## 🏷️ 版本号: v0.2.5

BackDesk `v0.2.5` 是一次聚焦稳定性、误触防护和可恢复性的维护更新。这个版本围绕长期运行后监听失效、截图工具穿透、Dock 两侧空白无法触发、以及极端情况下鼠标左键被吞等真实使用问题，补齐了自恢复、诊断日志和紧急兜底能力。

---

## 🚀 新增功能与深度修复

### 1. 🛡️ EventTap 自恢复与安全保险丝
*   **问题**：macOS 可能在回调超时、睡眠唤醒或输入系统状态变化后禁用 `CGEventTap`，表现为菜单栏 App 仍在但点击功能失效。
*   **解决**：新增 `tapDisabledByTimeout` / `tapDisabledByUserInput` 自恢复处理，并增加 watchdog 定期检查监听端口。若短时间内连续吞掉过多左键事件，会自动暂停监听 60 秒，避免鼠标左键大面积不可用。

### 2. 🖱️ 紧急暂停与调试日志
*   新增「高级兼容性」菜单，将兼容模式、点击调试日志和紧急暂停统一收纳。
*   新增「紧急暂停监听 5 分钟」与「立即恢复监听」菜单项。
*   新增 `Control + Option + Command + 左键` 兜底手势，可在极端情况下暂停监听 60 秒。
*   日志迁移到标准路径 `~/Library/Application Support/BackDesk/backdesk.log`，便于长期定位问题。

### 3. 📸 截图、选区与高层遮罩防穿透
*   **问题**：微信等截图工具启动后，点击截图工具按钮可能穿透到桌面，误触发显示桌面或 Mission Control。
*   **解决**：新增截图/选区类高层遮罩预检，在 AX 判定前先识别微信、企业微信、QQ、系统截图、Snipaste、CleanShot、Shottr、Xnip 及飞书/钉钉等常见工具，命中后放行给前台工具处理。

### 4. 🧭 Dock 两侧空白区域恢复触发
*   **问题**：旧逻辑将 Dock 的整条 `visibleFrame` 预留区域都当成 Dock，导致 Dock 两侧空白无法单击/双击触发。
*   **解决**：改为先判断 Dock 预留边缘，再只拦截 Dock 进程真实窗口范围。Dock 两侧空白现在会直接按桌面空白处理。

### 5. 🧩 应用兼容模式与拖拽保护
*   新增按当前前台 App 的兼容模式，适合会议、远控、截图、游戏、绘图等不希望 BackDesk 介入的场景。
*   新增拖拽取消逻辑，鼠标按下后移动超过阈值会取消挂起的单击动作，避免拖选、拖文件或截图选区时误触发。
*   增加窗口列表短缓存，减少 EventTap 回调中的重复系统查询。

## 🏷️ 版本号: v0.2.3

BackDesk `v0.2.3` 是一次具有重大里程碑意义的跨版本兼容与防御机制更新。我们成功实现了对 **macOS 14.0+ (Sonoma)** 新系统的完美防穿透支撑，同时为 **macOS 13.0 及以下系统** 提供了高保真的防误触优化，打造了无缝契合的多版本并行运行架构。

---

## 🚀 新增功能与深度修复

### 1. 📂 macOS 14.0+ (Sonoma) Finder 文件夹空白分栏防穿透
*   **问题**：Sonoma 系统引入了对 `CGWindowListCopyWindowInfo` 中窗口标题（`kCGWindowName`）的跨进程隐私封锁（不具有录屏权限的应用均获取为空）。这导致原有的几何检测失效，在 Finder **分栏视图 (Column View)** 点击最右侧空白列区域时，会错误穿透到壁纸，触发显示桌面。
*   **解决**：全新引入 **Accessibility API (辅助功能 API) 链式父级探测**。当鼠标点击落在 Finder 进程的元素上时，我们顺着 AX 节点链向上爬升，如果在其容器链中找到了子角色为标准实体窗口（`AXStandardWindow`）的祖先，则**百分百确认为实体窗口空白区域并安全拦截放行**，杜绝任何穿透！

### 2. 🛡️ macOS 13.0 及以下系统全场景防误触 (Dock/菜单栏/桌面图标)
*   **问题**：由于老系统（macOS 12/13）的桌面背景窗口在 Accessibility 中也被系统标记为了 `AXStandardWindow`，简单使用 AX 检查会导致点击空白壁纸时被错误拦截（软件完全失效）。为此退回 0.1.0 纯几何计算后，又引发了**点击 Dock 栏、顶部菜单栏、以及桌面文件/文件夹图标时频繁误触显示桌面**的历史遗留问题。
*   **解决**：我们为老系统重构并升级了**高精度三合一拦截机制**，并将老系统的广播监听完美接入新检测器：
    *   **顶部菜单栏拦截**：在 Cocoa 坐标转换层通过屏幕绝对边界对最顶部窄条进行高精度拦截。
    *   **物理 Dock 栏拦截**：通过 visibleFrame 和 screenFrame 的绝对数学差集，计算出 Dock 栏物理像素渲染盒，精准剔除 Dock 两侧空白与废纸篓/堆栈的任何点击。
    *   **桌面图标/文件拦截**：点击桌面文件时，AX API 能精确嗅探出点击元素为 `AXImage`/`AXStaticText` 等文件属性，且窗口**没有关闭按钮（`kAXCloseButtonAttribute`）且标题属于桌面黑名单**，从而安全拦截防误触。

### 3. 🚦 版本运行时双路隔离架构 (Dual-Path Architecture)
*   为了确保 macOS 14+ 现代拦截特性与 macOS 13- 极速稳定特性的双重绝对安全，我们构建了严格的分流架构：
    *   **监听分流**：macOS 14+ 启用 `CGEventTap` 实现“主动屏蔽误触”，macOS 13- 启用 `NSEvent` 广播监听实现“零系统阻碍”。
    *   **触发分流**：macOS 14+ 使用 Sonoma 支持的 `NSWorkspace.shared.open` 代理唤醒 Mission Control，macOS 13- 使用 `Process` 直接对二进制发送 IPC 指令（避免 arguments 丢失），保证两端动画丝滑可靠！
