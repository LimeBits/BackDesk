# Security Policy

BackDesk depends on macOS Accessibility permission, so security and privacy reports are important.

## Reporting a Vulnerability

Please do not open a public issue for vulnerabilities or reports that include sensitive logs.

Use a private contact channel preferred by the maintainer, or open a minimal GitHub issue asking for a private disclosure path without including sensitive details.

## Scope

Relevant reports include:

- unexpected interception of clicks in other apps
- privacy leaks in diagnostic logs
- unsafe update behavior
- permission handling that could confuse users or weaken macOS protections

BackDesk does not auto-upload logs and does not install updates silently.
